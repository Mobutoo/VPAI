  <files>roles/n8n-provision/files/workflows/cf-brief-to-concept.json</files>
  <action>
Create the n8n workflow JSON file for the brief-to-concept pipeline (FLOW-01). The workflow handles steps 1-5 of the content production pipeline.

**Webhook trigger:** POST path `cf-brief-to-concept`, responseMode `responseNode`.

**Node 1 — Validate Secret (Code node):**
Parse incoming body. Validate `body.secret === $env.N8N_WEBHOOK_HMAC_SECRET`. Extract: `content_id`, `brand_id`, `title`, `format`, `brief` (object with objective, target, message, constraints).

**Node 2 — Read Brand Profile (HTTP Request):**
Call `http://localhost:5678/webhook/cf-read-content` to get current content state. This is an internal n8n webhook call (localhost, not external URL).

**Node 3 — Step 1: Enhance Brief (Code node with LLM):**
Call LiteLLM (`http://litellm:4000/v1/chat/completions`) with model `deepseek-v3` (eco — non-creative metadata). System prompt: "You are a content strategist for Instagram. Enhance this brief with specific KPIs, audience insights, and hook angles." Include brand tone from input. Max tokens 1500, temperature 0.7.

**Node 4 — Step 2: Research (Code node with LLM):**
Call LiteLLM with model `deepseek-v3` (eco). System prompt: "Research trending topics and competitor content related to this brief. Output structured insights." Max tokens 2000, temperature 0.5.

**Node 5 — Step 3: Moodboard Prompt (Code node with LLM):**
Call LiteLLM with model `deepseek-v3` (eco). Generate 4-6 image prompts for moodboard based on brief + research. Output as JSON array of prompt strings.

**Node 6 — Step 4: Concept & Hook (Code node with LLM):**
Call LiteLLM with model `gpt-4o-mini` (creative). System prompt: "You are a viral content creator. Write a concept document with: hook (first 3 seconds), narrative arc, visual style, CTA. Brand tone: [from brand profile]." Max tokens 2000, temperature 0.8.

**Node 7 — Step 5: Casting (Code node with LLM):**
Call LiteLLM with model `deepseek-v3` (eco). Based on concept, define casting: characters/personas needed, voice requirements, visual descriptions. Max tokens 1000, temperature 0.5.

**Node 8 — Store Results (Code node):**
Call `http://localhost:5678/webhook/cf-update-content` with action `update_content`, passing all step results as data fields. Set `current_step: 5`, `status: "awaiting_validation"`.

**Node 9 — Sync Kitsu (HTTP Request):**
Call `http://localhost:5678/webhook/cf-kitsu-sync` with action `update_task_status`, setting steps 1-5 tasks to "done".

**Node 10 — Sync Calendar (HTTP Request):**
Call `http://localhost:5678/webhook/cf-calendar-sync` with action `sync_content` and the content_id. This auto-creates a Plane work item for this content (CAL-03 requirement). Only called on first execution (new content), not on re-runs.

**Node 11 — Webhook Response:**
Return `{ status: "ok", content_id, steps_completed: [1,2,3,4,5], concept_summary: "..." }`.

**Connections:** Linear chain: Trigger -> Validate -> Read -> Step1 -> Step2 -> Step3 -> Step4 -> Step5 -> Store -> Kitsu -> Calendar -> Response.

**Important details:**
- All Code nodes use `$http.request()` (NOT fetch or axios)
- Use unique UUIDs for each node ID (format: `btc-XXXX-...`)
- Each LLM Code node must handle errors gracefully (try/catch, return error in response)
- All internal webhook calls use `http://localhost:5678/webhook/` prefix (n8n calls itself internally)
- Tag the workflow with `content-factory`

**Webhook trigger:** POST path `cf-script-to-storyboard`, responseMode `responseNode`.

**Node 1 — Validate Secret (Code node):**
Parse body, validate secret. Extract: `content_id`. Read full content state via `http://localhost:5678/webhook/cf-read-content`.

**Node 2 — Step 6: Write Script (Code node with LLM):**
Call LiteLLM with model `gpt-4o-mini` (creative — this is the main creative writing step). System prompt: "You are a professional scriptwriter for short-form Instagram video content. Write a scene-by-scene script based on the concept document. Each scene must have: scene_number, duration_seconds, action, dialogue (if any), visual_description, transition. Output as JSON array." Include concept, hook, casting from content data. Max tokens 4000, temperature 0.8.

**Node 3 — Step 7: Storyboard Decomposition (Code node):**
Parse the script JSON. For each scene, call `http://localhost:5678/webhook/cf-scene` with action `create_scene` to create NocoDB scene rows with fields: `content_id`, `scene_number`, `description` (visual_description), `duration`, `dialogue`, `transition`, `status: "pending"`, `provider: ""` (to be assigned by generate-assets).

**Node 4 — Step 8: Sound Design Brief (Code node with LLM):**
Call LiteLLM with model `deepseek-v3` (eco). Generate sound design notes: music style, SFX list per scene, voice-over cues. Max tokens 1500, temperature 0.5.

**Node 5 — Store Results (Code node):**
Call `http://localhost:5678/webhook/cf-update-content` with all step data. Set `current_step: 8`, `status: "awaiting_validation"`.

**Node 6 — Sync Kitsu (HTTP Request):**
Call `http://localhost:5678/webhook/cf-kitsu-sync` with action `update_task_status`, steps 6-8 -> "done".

Return `{ status: "ok", content_id, scenes_created: N, steps_completed: [6,7,8] }`.

**Connections:** Linear chain: Trigger -> Validate -> Script -> Storyboard -> SoundDesign -> Store -> Kitsu -> Response.

**Important:** Scene creation in Node 3 must loop over each scene from the script output. Use a for loop inside the Code node, calling the cf-scene webhook for each scene sequentially.
