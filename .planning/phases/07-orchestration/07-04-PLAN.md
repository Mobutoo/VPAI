Create the n8n workflow JSON for Plane editorial calendar sync (CAL-01, CAL-02, CAL-03).

**Webhook trigger:** POST path `cf-calendar-sync`, responseMode `responseNode`.

**Node 1 — Validate and Route (Code node):**
Validate secret. Extract: `action` (one of: `create_work_item`, `create_cycle`, `add_to_cycle`, `sync_content`), `data` object.

**Node 2 — Route Action (Switch node):**
Route to appropriate handler based on action.

**Node 3a — Create Work Item (Code node):**
Call Plane API to create a work item for a content entry:
```javascript
const workItem = await $http.request({
  method: 'POST',
  url: `${$env.PLANE_PUBLIC_URL}/api/v1/workspaces/ewutelo/projects/e0cb95f0-0ea5-41b8-a3e3-aec45e8cc37e/work-items/`,
  headers: {
    'X-API-Key': $env.PLANE_API_KEY,
    'Content-Type': 'application/json'
  },
  body: {
    name: data.title,
    description_html: `<p>Format: ${data.format}</p><p>Brief: ${data.brief_summary}</p>`,
    start_date: data.created_date,
    target_date: data.publish_date,
    module: 'c04ac29e-9842-4eec-8ff6-6923e9fe75d7'  // CF module
  }
});
```

**Node 3b — Create Cycle (Code node):**
Create a Plane cycle representing a content drop:
```javascript
const cycle = await $http.request({
  method: 'POST',
  url: `${$env.PLANE_PUBLIC_URL}/api/v1/workspaces/ewutelo/projects/e0cb95f0-0ea5-41b8-a3e3-aec45e8cc37e/cycles/`,
  headers: { 'X-API-Key': $env.PLANE_API_KEY, 'Content-Type': 'application/json' },
  body: { name: data.cycle_name, start_date: data.start_date, end_date: data.end_date }
});
```

**Node 3c — Add to Cycle (Code node):**
Add work items to a cycle:
```javascript
await $http.request({
  method: 'POST',
  url: `${$env.PLANE_PUBLIC_URL}/api/v1/workspaces/ewutelo/projects/e0cb95f0-0ea5-41b8-a3e3-aec45e8cc37e/cycles/${data.cycle_id}/cycle-issues/`,
  headers: { 'X-API-Key': $env.PLANE_API_KEY, 'Content-Type': 'application/json' },
  body: { issues: data.work_item_ids }
});
```

**Node 3d — Sync Content (Code node):**
Full sync: read content from NocoDB via cf-read-content, create or update corresponding Plane work item. This is the action triggered by NocoDB webhook on row creation.

First check if work item already exists (search by name). If exists, update. If not, create new.

**Node 4 — Webhook Response:**
Return `{ status: "ok", action, plane_response: ... }`.

**Connections:**
- Trigger -> Validate -> Switch
- Switch output 0 -> CreateWorkItem -> Response
- Switch output 1 -> CreateCycle -> Response
- Switch output 2 -> AddToCycle -> Response
- Switch output 3 -> SyncContent -> Response
