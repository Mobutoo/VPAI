**roles/kitsu/templates/supervisord.conf.j2:**
CRITICAL: This overrides the default supervisord.conf to:
1. Remove DB_USERNAME and DB_PASSWORD from the gunicorn environment (so Docker env vars flow through)
2. Disable internal PostgreSQL (autostart=false) — use external shared PG
3. Disable internal Redis (autostart=false) — use external shared Redis

```ini
pidfile=/var/run/supervisord.pid

autorestart=true
stdout_logfile=NONE
stderr_logfile=NONE
stdout_events_enabled=true
stderr_events_enabled=true

[program:gunicorn-events]

directory=/opt/zou
autostart=true
autorestart=true
stdout_logfile=NONE
stderr_logfile=NONE

stderr_events_enabled=true

[program:nginx]

result_handler=supervisor_stdout:event_handler
```

**5. roles/docker-stack/templates/docker-compose.yml.j2** — Add Kitsu service block in the APPLICATION LAYER section (after existing services like nocodb, before monitoring section). Add between the last application service and monitoring:
```yaml
  # === KITSU (CGWire Production Tracker) ===
  kitsu:
    image: {{ kitsu_image }}
    container_name: {{ project_name }}_kitsu
    restart: unless-stopped
    security_opt:
      - no-new-privileges:true
    cap_drop:
      - ALL
    cap_add:
      - CHOWN
      - SETGID
      - SETUID
      - DAC_OVERRIDE
      - FOWNER
    env_file:
      - /opt/{{ project_name }}/configs/kitsu/kitsu.env
    volumes:
      - {{ kitsu_previews_dir | default('/opt/' + project_name + '/data/kitsu/previews') }}:/opt/zou/previews
      - {{ kitsu_config_dir | default('/opt/' + project_name + '/configs/kitsu') }}/supervisord.conf:/etc/supervisord.conf:ro
    networks:
      - backend
    deploy:
      resources:
        limits:
          memory: {{ kitsu_memory_limit | default('512M') }}
          cpus: "{{ kitsu_cpu_limit | default('1.0') }}"
        reservations:
          memory: {{ kitsu_memory_reservation | default('256M') }}
    healthcheck:
      test: ["CMD-SHELL", "curl -sf http://127.0.0.1/api/health || exit 1"]
      interval: 30s
      timeout: 10s
      retries: 5
      start_period: 120s
```

**NOTE on healthcheck (INFRA-05):** The `/api/health` endpoint is a shallow check — it confirms nginx+gunicorn are running but does NOT verify DB connectivity. The Kitsu UI may show "API unreachable" even when the healthcheck passes if supervisord misconfiguration prevents Zou from reaching PostgreSQL. This is an accepted limitation for Docker-level liveness. Deep DB connectivity is verified during provisioning (Plan 03) via `zou upgrade-db`. For Grafana monitoring, cAdvisor autodiscovery handles Kitsu — no Alloy config changes needed. The `discovery.docker "containers"` block in `config.alloy.j2` already scrapes ALL Docker containers, so `{{ project_name }}_kitsu` will appear in cAdvisor/Grafana metrics (CPU, memory, network, restarts) automatically.
