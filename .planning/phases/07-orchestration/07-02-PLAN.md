  <files>roles/n8n-provision/files/workflows/cf-generate-assets.json</files>
  <action>
Create the n8n workflow JSON for per-scene asset generation (FLOW-04). This workflow reads all scenes for a content, assigns a provider to each, and dispatches via the existing creative-pipeline webhook.

**Webhook trigger:** POST path `cf-generate-assets`, responseMode `responseNode`.

**Node 1 — Validate and Load (Code node):**
Validate secret. Extract `content_id`. Call `http://localhost:5678/webhook/cf-read-content` to get content metadata (format, brand). Call `http://localhost:5678/webhook/cf-scene` with action `list_scenes` to get all scenes.

**Node 2 — Assign Providers (Code node):**
For each scene, determine the provider based on scene type:
- If scene has `visual_description` containing motion/animation keywords -> `model: "remotion"`, `type: "video-composition"`
- If scene has static image description -> `model: "comfyui"`, `type: "image"` (local first)
- Default fallback: `model: "seedream"`, `type: "image"` (cloud via LiteLLM)

Map the content `format` field to Remotion composition name:
- `reel-motion-text` -> `ReelMotionText`
- `reel-meme-skit` -> `ReelMemeSkit`
- `reel-feature-showcase` -> `ReelFeatureShowcase`
- `reel-teaser` -> `ReelTeaser`

Output: array of scene dispatch objects with provider assignment.

**Node 3 — Dispatch to Creative Pipeline (Code node with loop):**
For each scene, call `http://localhost:5678/webhook/creative-pipeline` with:
```
{
  "type": scene.assigned_type,
  "prompt": scene.visual_description,
  "model": scene.assigned_model,
  "composition": compositionName,  // only for video-composition type
  "input_props": { "scenes": [scene], "brand": brandData },
  "output_name": `${content_id}-scene-${scene.scene_number}`,
  "agent_id": "content-factory"
}
```
Collect results (result_url, render_id, provider) for each scene.

**IMPORTANT:** Use sequential dispatch (not parallel) to avoid overwhelming local ComfyUI/Remotion. Add 2-second delay between dispatches via `await new Promise(r => setTimeout(r, 2000))`.

**Node 4 — Update Scene Status (Code node):**
For each completed scene dispatch, call `http://localhost:5678/webhook/cf-scene` with action `update_scene` to store `asset_url`, `provider`, `status: "generated"`.

**Node 5 — Sync Kitsu (Code node):**
Call `http://localhost:5678/webhook/cf-kitsu-sync` with action `update_task_status`, step 9 -> "done".

**Node 6 — Webhook Response:**
Return `{ status: "ok", content_id, scenes_dispatched: N, results: [...] }`.

**Connections:** Linear: Trigger -> Validate -> Assign -> Dispatch -> UpdateScenes -> Kitsu -> Response.

  <files>roles/n8n-provision/files/workflows/cf-rough-cut.json</files>
  <action>
Create the n8n workflow JSON for rough-cut video assembly via Remotion (FLOW-05). This workflow collects all scene assets and triggers a full composition render.

**Webhook trigger:** POST path `cf-rough-cut`, responseMode `responseNode`.

**Node 1 — Validate and Load (Code node):**
Validate secret. Extract `content_id`. Call cf-read-content to get format, brand data. Call cf-scene with action `list_scenes` to get all scenes with their asset URLs.

**Node 2 — Validate All Assets Ready (Code node):**
Check that ALL scenes have `status: "generated"` and valid `asset_url`. If any scene is missing or pending, return error response: `{ status: "error", message: "Scenes not ready", pending_scenes: [...] }`.

**Node 3 — Build Remotion Props (Code node):**
Construct the ReelProps object for the Remotion render:
```javascript
const composition = formatToComposition(content.format);
// Map format to composition name
const reelProps = {
  scenes: scenes.map(s => ({
    id: `scene-${s.scene_number}`,
    type: s.asset_url.endsWith('.mp4') ? 'video' : 'image',
    src: s.asset_url,
    durationInFrames: Math.round(s.duration * 30), // 30fps
    text: s.dialogue || '',
    transition: s.transition || 'fade'
  })),
  brand: {
    name: content.brand_name,
    primaryColor: content.brand_palette?.primary || '#FF6B00',
    font: content.brand_typography || 'Inter'
  }
};
```

**Node 4 — Trigger Remotion Render (Code node):**
Call `http://localhost:5678/webhook/creative-pipeline` with:
```
{
  "type": "video-composition",
  "prompt": content.title,
  "model": "remotion",
  "composition": composition,
  "input_props": reelProps,
  "output_name": `${content_id}-rough-cut`,
  "agent_id": "content-factory"
}
```
Store the `render_id` and `result_url` from response.

**Node 5 — Store Rough Cut URL (Code node):**
Call cf-update-content to store rough_cut_url, set `current_step: 10`, `status: "awaiting_validation"`.

**Node 6 — Sync Kitsu + Upload Preview (Code node):**
Call cf-kitsu-sync with action `update_task_status` (step 10 -> "done") and action `upload_preview` with the render URL.

**Node 7 — Webhook Response:**
Return `{ status: "ok", content_id, render_id, rough_cut_url, composition }`.

**Connections:** Linear: Trigger -> Validate -> CheckAssets -> BuildProps -> Render -> Store -> Kitsu -> Response.

**Error handling:** If Remotion render fails (non-200 response), return error with render details. Set workflow timeout to 600000ms (10 min) in settings: `"executionTimeout": 600`.
